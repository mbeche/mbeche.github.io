Sprint 4: Create interface for making new accounts

April 28, 2018 at 9:22pm

The tutorial I was using for Sprint 3 by William S. Vincent has a followup tutorial Django User Signup to make a user account setup app. Apparently Django does not provide views for this, so we will be creating the interface in a new app. It will be interesting to integrate this with my project!

After completing this sprint, I will go back and work on the post/comments template and model, then pretty things up with HTML formatting and CSS:

  • Sprint 5: Create the models, templates and views for posts and comments
  • Sprint 6: Use CSS to make things look nice

Breakdown of Sprint 4:

  1. create an app: python manage.py startapp accounts
  2. Add app to the INSTALLED_APPS in the settings.py file (The author of the tutorial just has 'accounts' added to the list. I would expect that 'accounts.apps.accountsConfig' to be the entry we should add. We'll see if the author's approach works and then I'll try mine if it doesn't.)
  3. Create a URLconf for accounts. This will have the same path as the accounts URLconf we created in the last sprint and will need to go before that URLconf. This is because Django looks top to bottom for URLconf. So it'll see our new URLconf within our accounts app, and then (or otherwise) push forward to the normal login if an account is created. (how does the system know if we have an account created or not? hmm..)
  4. Create a urls.py file inside the accounts directory for signup
  5. Go into views and create a SignUp class where we use the built in UserCreationForm. We also use a reverse_lazy('login') to redirect the user to the login page once they have created an account. (The reverse lazy has an explanation but the rational eludes me at the moment. More to be revealed I suppose.)
  6. Create a template: 'signup.html' in the project (imtwitter) level templates folder
    1. Why the project level and not app level? Still trying to figure this out
    2. Use the {% extends 'base.html' %} format again.
    3. Continued point of confusion:The html looks exactly the same as the 'login.html' with {% csrf_token %} and {{ form.as_p }}. How does Django know what form to use?

That was painless. Currently the directory the account creation lives at is: http://127.0.0.1:8000/accounts/signup/
I think that I need to add a link to that on the main page as an alternative to login.
