Sprint 3: Making User Login Interface

April 28, 2018 at 6:21pm

After reading through the Django Login/Logout Tutorial by William S. Vincent I have decided on breaking out into some smaller sprints:

  • Sprint 3: Create the login/logout dynamics
  • Sprint 4: Create the new account user interface
  • Sprint 5: Create the templates and views for posts and comments
  • SPrint 6: Use CSS to make things look nice

Breakdown of Sprint 3:

  1. No need to install anything 'django.contrib.auth' is automatically included in the INSTALLED_APPS under settings.py
  2. Create urlpatterns with 'django.contrib.auth.url' for accounts URL. This contains quite a few Authentication Views
    3. I discover after writing out the following steps that the tutorial uses a project without an app, whereas my project has an app. A comment suggests we should create the templates and URLs within the main project folder, not the app folder. I will stick with that for now.
  3. Create a login page
    1. Django by default looks in 'templates/registration/' for auth templates
    2. Create 'login.html' file in this template directory
    3. Enter html code into this file
    4. The code contains a form element with a method attribute of "post", as well as {$ csrf_token %} and {{ form.as_p}} and a login button. I understand that the former helps mitigate security risks and the latter wraps each input in a paragraph, however I don't understand where the from content comes from. The code somehow knows to supply the 'login' and 'password' fields. Maybe it will click once I actually implement it on my site.
    5. My best guess is that Django must know login.html corresponds to the 'login' view
  4. Update 'settings.py' to have Django look for templates within the imtwitter directory
  5. Set LOGIN_REDIRECT_URL = '/' to set the login reditect to the homepage
    It worked! :D
    I had trouble because for some reason my project is an extra folder so I had to change the DIRS from 'templates' to 'imtwitter/templates'. Thanks to the debugging tools and checking my local directory it was pretty easy to figure out.
  6. Create more superusers to try logging in (I should be able to log in with my current superuser account)
  7. Create template for homepage in the templates directory (The tutorial directs to create home.html and base.html. I'm not completely certain why this is the case but I think it should make sense as I complete the tutorial. I currently don't have any templates, I am using HttpResponse to put some text on my home page.)
  8. Import TemplateView URLconf (Just to spice things up, I did the last two steps in the IMTwitter folder. It seems to be working out alright!)
  9. Create a link for logging out on within the template
  10. Add LOGOUT_REDIRECT_URL to redirect to the homepage as well
  11. There's some shifting around of the views and urls that I'm not certain about, however I think it will make sense as I complete the tutorial.

Wow, creating the login/logout interface was much simpler than I expected. Talking with the Python group today it sounded like it would have been much more of hassle. Of course, I still need to create the account creation app, this might be a bit more challenging!

I also learned a bit about how Django {% extends 'base.html' %} and {% block content %} {% endblock %} works. This will allow me to have the same template HTML file and a variety of different bodies that I specify in other HTML files. Pretty nifty!

I'm excited to move on with creating the account creation app. It's awesome to see all of these pieces coming together, but I know I have plenty of work past this. To summarize-
