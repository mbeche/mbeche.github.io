Error While Deploying

I decided to try a local test on whether my project would boot up for you. I created a new DB and copied my project into a new folder. I entered my new DB authentication in settings.py and tested migrate/makemigrations. I expected no errors since migrations had been made before copying. I then received the error below:

django.db.utils.ProgrammingError: relation "auth_user" does not exist
LINE 1: ...user"."is_active", "auth_user"."date_joined" FROM "auth_user...

This is the same error I received when trying to deploy my project online with DigitalOcean.


I hope that it's user error and I am forgetting a step to get my database set up correctly when transferring my project to a new DB, but I thought I would throw this note in just in case you came across this issue, too.

To fix the issue on DigitalOcean, I created a new project, made initial migrations for the user model, and then copyied my app folders over. I also had to change settings.py to add these apps, static file dir, and login/logout redirects.
